backbone.js

  • +Структурує код;
  • +Тривалий час популярний;
  • +Мінімалістичний;
  • +Свобода рішень;
  • -Свобода рішень.
Легковісний фреймворк, створений в 2010-у році. Став швидко популярний у противагу монстрам типу ExtJS, які тоді домінували на ринку. Вікористовується нині такими ресурсами, як Pinterest та AirBNB. Структурує код - історія про iMallTool.

angular.js

  • +Швидке прототипування;
  • +Велика увага тестуванню;
  • +Easy to learn, difficult to master;
  • +Привильне місце для HTML. У представленні;
  • -Швидкість/Марнотратність;
  • -Весь у собі;
  • -"Стихійний" дизайн.
Створенний у 2009-у році. Розробники, для прикладу переписали застосунок зо скаладвся з 17 тисяч строк коду, та був написаний за 6 місяців. Переписали за три тижні і вклалися в тисячу строк. Чим привернули неабияку увагу спільноти і в першу чергу Google, яка і взяла framework під своє крило. Активно використовується в Google та Virgin. Швидкість/Марнотратність - повільні наглядачі (watches).

ember.js

  • +Вирішує частину проблем Angular-y;
  • -Створює ще більше проблем;
  • +Чіткі залежності;
  • -Багато коду, для базового функціоналу сторінки;
  • +Набір конвенцій;
  • -Сильно переписувався декілька разів. Високий порог входження.
З 2007-го року розроблявся різними людьми та компаніями, в тому числі і Apple. Потім був форкунутий core contributor-ом таких проектів, як Rails и jQuery - Yehuda Katz. Використовуєтья в Yahoo! і Groupon.

$(document).ready(function() {
  var Currency = Backbone.Model.extend({
    defaults: {
      dollars: 0
    },
    hryvnias: function(value) {
      return this.get('dollars') * kurs;
    }
  })

  var ConverterView = Backbone.View.extend({
    el: document.getElementById('backbone'),
    model: new Currency(),
    events: {
      'input #backboneDollar': 'updateDollars',
    },
    initialize: function() {
      this.listenTo(this.model, 'change:dollars', this.render)
      this.render()
    },
    render: function() {
      this.$('#backboneDollar').val(this.model.get('dollars'))
      this.$('#backboneHryvnia').val(this.model.hryvnias())
    },
    updateDollars: function(event) {
      this.model.set('dollars', event.target.value)
    },
  })

  var converterView = new ConverterView();
});

Ідея повного розділення frontend та backend.

Всі клієнти рівноцінні. І мобильні і веб.

  • -Складний або неможливий рендерінг HTML на стороні backend.
  • -Single Page Application парадигма ломає існуючий уклад вебу.
  • -Складність використання, як рядових компонентів сторінки.

У цій сфері ще буде етап встановлення стандартів, а на даний момент ми маємо багато невирішенних проблем.
